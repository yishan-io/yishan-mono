## 为什么需要移山？

每次启动新的 Agent 会话，你都要重复解释项目结构、约定和已有决策——解释的负担全压在你身上。

移山为 Agent 提供持久的项目记忆、共享上下文和可复用的工作流程。Agent 自己找到所需信息，你不再需要反复解释。

## 核心功能

- **独立工作区**：一个任务，一个工作区。每个 Agent 拥有独立的分支、终端和文件状态。任务切换，什么都不会丢。
- **随项目成长的记忆**：`.my-context` 和内置记忆自动积累决策、笔记和会话摘要。Agent 自己找上下文。
- **消除繁琐仪式的技能**：把你的工作方式定义一次——审查模式、测试规则、交接步骤。每个 Agent 每次都按同一套流程来。

## 更多

- **定时任务**：按计划触发定时 Agent 任务——代码审查、健康检查、周报汇总——在你专注其他事情时自动运行。
- **PR 状态**：不离开移山，直接查看 Pull Request 状态、CI 检查结果和 Review 进度。
- **兼容你的工具**：在 VS Code、Cursor、JetBrains、Sublime、Xcode 等编辑器中打开文件。移山是桥梁，不是替代。
- **带上你的 Agent**：兼容 OpenCode、Claude、Codex、Gemini、Cursor、Pi 等。用你已经熟悉的 Agent。

## 路线图

- 远程主机工作空间：在远程机器上启动工作流 *(开发中)*
- CLI + AI 工作流整合：从聊天或命令行创建多个工作空间 *(开发中)*
- 开发生命周期管理：从 Issue 到工作空间再到 PR 的完整流程 *(规划中)*
- 移动端远程控制：通过手机监控和操控工作空间 *(规划中)*
