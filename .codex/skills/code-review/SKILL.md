# Code Review

Run a structured code review with severity and confidence filters.
The implementation is shared in `scripts/automation/collect-review-context.sh` so all agents can reuse the same context collection.

## Workflow

1. Define review scope.
- Prefer `base-ref...head-ref` scope.
- If user gives no scope, default to current branch against `origin/main`.

2. Collect review context.
- Read `AGENTS.md`, relevant module docs, and changed files.
- Run `scripts/automation/collect-review-context.sh [base-ref] [head-ref]` to gather commits + changed file list.

3. Review in passes.
- Correctness and regressions.
- Security and data safety.
- Test coverage and edge cases.
- API/contract compatibility and migration risks.
- Documentation and operability impact.

4. Classify findings.
- `high`: likely bug/security/data-loss or breaking behavior.
- `medium`: meaningful risk or missing protection.
- `low`: maintainability/readability concerns.
- Track confidence (0-100). Report only findings with confidence >= 80 unless user asks for all.

5. Output format.
- Findings first, ordered by severity, each with:
  - file reference
  - problem
  - impact
  - suggested fix
  - confidence
- Then list open questions/assumptions.
- If no findings, say so explicitly and list residual risks/testing gaps.

## Commands

Collect review scope:

```bash
scripts/automation/collect-review-context.sh
```

Custom range:

```bash
scripts/automation/collect-review-context.sh origin/main HEAD
```
