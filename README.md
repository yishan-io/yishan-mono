**The workspace built for parallel minds.**

Agents made parallel work easy. Managing it is still on you. Yishan fixes that.

- **Every workstream gets a living workspace** — terminals, agent sessions, file state, and git context stay hot. Switch between them without reloading everything into your head.
- **One place to monitor all active work** — see what changed, what is running, and what needs attention across all workstreams at a glance.
- **Worktree-based isolation** — each task runs in its own git worktree so parallel work never collides. Yishan makes worktrees practical for daily use, not just an occasional CLI trick.
- **Context stays with the work** — built-in `.my-context` keeps notes, plans, and handoff details attached to each workspace, outside normal commit flow. Resume any task in seconds.
- **Agent-friendly by design** — use your existing agent tools (OpenCode, Claude, Gemini, etc.) inside each workspace. Agent work stays connected to the repo and terminal, not siloed in a separate chat tab.

## Key Capabilities

- Worktree-first workspace management — create, monitor, and switch between isolated branch workspaces
- Multi-pane dev environment — terminal, file editor, diff viewer, and browser side by side
- Git change tracking, branch visibility, and pull request awareness
- Open in VS Code, Cursor, JetBrains IDEs, Windsurf, Zed, or system file manager
- Multi-organization support with member roles and compute node management
- Scheduled agent automation and voice transcription
- In-app notifications and sound alerts for run completion and failures

## Works With Your Tools

Yishan works alongside the agent CLIs you already use — OpenCode, Codex, Claude, Gemini, Cursor Agent, Pi, and Copilot.
